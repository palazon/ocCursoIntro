10. Revisión asistida por pares

Author

JA Palazón Ferrando

Published

May 30, 2026

Objetivo

Aprender a usar OpenCode como revisor de documentos: detectar errores, incoherencias y oportunidades de mejora antes de entregar o publicar.

Duración estimada: 20 minutos

OpenCode como segundo par de ojos

Antes de entregar un documento, siempre viene bien que alguien lo revise. OpenCode puede hacer ese trabajo:

  • Detectar errores ortográficos y gramaticales
  • Señalar incoherencias entre secciones
  • Valorar el tono y la claridad del texto
  • Identificar lagunas de información

Como está en Plan, puedes pedir revisiones sin miedo a que modifique nada.

Manos a la obra

Ejercicio 1 — Revisión general

Plan Plan.

@darwin-evolucion.md Haz una revisión completa de este documento. Señala:

  1. Errores ortográficos o gramaticales
  2. Frases que podrían ser más claras
  3. Secciones que faltan o están incompletas
  4. Problemas de coherencia o estructura

Ejercicio 2 — Detectar contradicciones

@darwin-evolucion.md Lee todo el documento con atención. ¿Hay alguna contradicción entre lo que se dice en una sección y lo que se dice en otra? Si la hay, señálala.

Ejercicio 3 — Revisión de tono y audiencia

@trabajo-equipo.md ¿El tono de este documento es adecuado para un informe empresarial? Si no lo es, ¿qué cambiarías?

Ejercicio 4 — Aplicar correcciones (Build)

Pasa a Build Build.

@darwin-evolucion.md Aplica todas las correcciones que sugeriste en tu revisión. Corrige errores, mejora la claridad y reordena si es necesario.

Después de aplicarlas, vuelve a Plan y pide una segunda revisión para confirmar que todo está correcto.

Ejercicio 5 — Checklist de entrega (opcional)

@darwin-evolucion.md Dime si este documento cumple con estos requisitos para ser entregado:

  • Tiene un título claro
  • Tiene una introducción
  • Usa un tono coherente
  • No tiene errores ortográficos
  • Tiene una conclusión o cierre

Revisar en Plan, corregir en Build

Este flujo es muy útil y lo usarás a menudo:

  1. En Plan, pide una revisión sin riesgo
  2. Analiza las sugerencias
  3. Pasa a Build y aplica las que te parezcan bien
  4. Vuelve a Plan para una segunda vuelta si hace falta

Resumen del paso

  • ✅ Has pedido una revisión completa de un documento
  • ✅ Has detectado contradicciones entre secciones
  • ✅ Has valorado el tono para una audiencia concreta
  • ✅ Has aplicado correcciones y vuelto a revisar
  • ✅ Tienes un flujo de revisión en dos modos: Plan y Build

En el próximo paso veremos cómo compartir sesiones y preparar documentos para su entrega final.
