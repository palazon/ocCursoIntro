Más ideas para usar OpenCode en tu día a día

Documentación para novatos

OpenCode destaca especialmente en tareas de documentación. Aquí tienes ejemplos progresivos para quien empieza:

De la grabación al acta 🟢

Tomas notas sueltas durante una reunión y OpenCode las convierte en un acta profesional:

He apuntado: “reunión 10 jun — Pedro retraso en servidores — Laura presupuesto Q3 listo — acuerdo: reunión semanal los lunes — pendiente: migrar datos antes del 30”. Genera un acta con: asistentes, acuerdos, pendientes y próxima reunión.

Variación: pídele que genere el acta en tono formal para enviar por correo, o en tono informal para el equipo interno.

Sugerencia: crea un comando /acta-reunion para acelerar aún más el proceso.

De la idea al esquema 🟢

Describes un proyecto en pocas frases y OpenCode te devuelve un índice estructurado para empezar a escribir:

Necesito documentar el proceso de incorporación de nuevos empleados. Dame un esquema del manual con las secciones que debería incluir, ordenadas de forma lógica.

Una vez conforme, pídele que desarrolle cada sección.

Variación: pídele el esquema en formato tabla con tiempo estimado de redacción por sección.

Sugerencia: una vez conforme con el esquema, pídele que desarrolle cada sección en Build.

Del correo al documento formal 🟡

¿Tienes un correo informal explicando un proceso? Conviértelo en un procedimiento documentado:

`@correo-proceso.txt` A partir de este correo informal, genera un documento de procedimiento estándar con: objetivo, alcance, responsables, pasos detallados y referencias. Guárdalo como procedimiento-oficial.md.

Variación: pídele una versión imprimible o una infografía textual con los pasos resumidos.

Sugerencia: pídele también una versión resumida en una página para compartir con el equipo.

De notas dispersas a wiki unificada 🟡

Si tienes varios archivos sueltos con ideas, OpenCode los unifica en un documento coherente:

`@nota-1.md` `@nota-2.md` `@nota-3.md` Combina estos tres archivos en un solo documento bien estructurado. Elimina duplicados, ordena las ideas por tema y pon títulos a cada sección. Guárdalo como `wiki-unificada.md`.

Variación: pídele que añada un índice al principio y enlaces entre secciones relacionadas.

Sugerencia: añade un índice al principio y enlaces entre secciones para navegar mejor.

De técnico a divulgativo 🟡

Tomas un documento técnico y pides una versión para principiantes:

`@manual-tecnico.md` Reescribe este manual como si se lo explicaras a una persona que nunca ha usado el producto. Elimina jerga técnica, usa ejemplos cotidianos y divide la información en pasos muy claros. Guárdalo como `guia-principiantes.md`.

Variación: pídele también una versión “cheat sheet” de una sola página con lo esencial.

Sugerencia: pídele también una “cheat sheet” de una página con lo esencial para consulta rápida.

De transcripción a tutorial 🟡

¿Grabaste un vídeo formativo? Pasa la transcripción y OpenCode la convierte en un tutorial escrito:

`@transcripcion.txt` A partir de esta transcripción de vídeo, crea un tutorial paso a paso. Incluye: objetivo, materiales necesarios, pasos numerados con explicaciones y un resumen final. Guárdalo como `tutorial-escrito.md`.

Variación: pídele que extraiga solo los pasos prácticos (como una lista de acciones) o que genere una versión FAQ con las dudas más frecuentes.

Sugerencia: pídele que extraiga solo los pasos prácticos como lista de acciones rápida.

Planificación y calendarios

Programar actividades y generar ICS 🟡

OpenCode puede generar un archivo de calendario (.ics) que puedes importar en Google Calendar, Outlook o cualquier aplicación de calendario. Solo tienes que describir las actividades:

Soy profesor y necesito un calendario con las siguientes actividades:

  • Examen parcial: 15 de octubre, 10:00-12:00
  • Entrega de prácticas: cada viernes hasta el 30 de noviembre
  • Tutorías: lunes y miércoles de 16:00 a 18:00

Genera un archivo .ics con todos estos eventos. Guárdalo como calendario-curso.ics.

Variación: pídele eventos recurrentes (clases semanales, reuniones mensuales) o importa fechas desde una lista en Markdown.

Sugerencia: usa Plan para definir las actividades y Build para generar el archivo .ics.

Proyectos de varios pasos 🟡

Para proyectos grandes, OpenCode puede dividirlos en hitos y tareas, generando un plan ejecutable paso a paso:

Necesito organizar la mudanza de oficina. Tengo 3 meses por delante. Dame un plan con hitos mensuales, tareas semanales y responsables para cada una. Guárdalo como `plan-mudanza-oficina.md`.

El resultado es un documento con fechas, dependencias y checklists que puedes ir marcando a medida que avanzas.

Variación: pídele un diagrama de Gantt en tabla Markdown o una estimación de tiempos para cada fase del proyecto.

Sugerencia: divide el proyecto en hitos mensuales y usa @ para referenciar el plan en cada fase.

Compras y finanzas

Análisis de productos 🟢

Cuando dudes entre varios productos, pídele a OpenCode una comparativa con las características que realmente importan:

Voy a comprar un portátil para desarrollo. Compara estos modelos:

  • Modelo A: i7, 16 GB RAM, 512 GB SSD, 950 €
  • Modelo B: i5, 32 GB RAM, 1 TB SSD, 1100 €
  • Modelo C: Ryzen 7, 16 GB RAM, 512 GB SSD, 850 €

Crea una tabla comparativa con las características relevantes para programación y dime cuál ofrece mejor relación calidad-precio.

Variación: pídele un análisis según tu uso concreto (edición de vídeo, gaming, ofimática) o que incluya opiniones de fuentes especializadas. Usa el enlace a la página del proyecto.

Sugerencia: incluye siempre tu caso de uso concreto para que la comparativa sea relevante.

Comparación de ofertas económicas 🟡

Si tienes varios presupuestos de distintos proveedores, OpenCode los analiza por ti:

`@presupuesto-A.pdf` `@presupuesto-B.pdf` `@presupuesto-C.pdf` Compara estas tres ofertas. Dime: 1. Cuál es más barata en precio total 2. Cuál incluye más servicios 3. Diferencias en plazos de entrega y condiciones de pago 4. Cuál recomendarías y por qué

Variación: si los presupuestos están en texto, pégalos directamente en el chat. Pídele también un resumen ejecutivo para presentar a tu equipo.

Sugerencia: pídele también un resumen ejecutivo para presentar a tu equipo.

Alimentación

Guía de platos favoritos 🟢

OpenCode puede ayudarte a recopilar y organizar tus recetas preferidas en un documento estructurado:

Crea una guía con mis 10 platos favoritos. Para cada uno incluye: nombre, ingredientes, tiempo de preparación, nivel de dificultad y un consejo personal. Guárdala como `mis-platos-favoritos.md`.

Luego puedes ampliarla por categorías: primeros, segundos, postres, o por tipo de cocina.

Variación: pídele que añada fotos descriptivas (texto alternativo) o que calcule el coste aproximado por ración o el aporte calórico, …

Sugerencia: amplíala por categorías o por tipo de cocina para tener un recetario completo.

Programación dietética personalizada 🟡

A partir de tus platos favoritos y necesidades nutricionales, OpenCode te genera un plan semanal equilibrado:

`@mis-platos-favoritos.md` A partir de mis recetas, crea una programación semanal de comidas equilibrada. Incluye desayuno, comida y cena para 7 días. Asegúrate de variar los grupos de alimentos y evitar repetir el mismo plato dos días seguidos. Añade la lista de la compra al final.

Variación: pídele adaptaciones para objetivos concretos: pérdida de peso, ganancia muscular, dieta baja en hidratos, o menú semanal para toda la familia.

Sugerencia: usa @ para enlazar con tu recetario y generar la lista de la compra automáticamente cada semana.

Sistema y mantenimiento

Diagnóstico de errores 🟡

Cuando tu sistema da un error inesperado, puedes copiar el mensaje y pedirle a OpenCode que lo analice:

He visto este error al arrancar el programa: `Fallo al cargar librería libXYZ.so`. ¿Qué puede estar causándolo? Dame causas ordenadas por probabilidad y pasos para solucionar cada una.

Variación: pídele un informe con recomendaciones para evitar que el error se repita o que compare el error con otros conocidos.

Sugerencia: incluye el sistema operativo, la versión del software y qué estabas haciendo cuando ocurrió el error.

Plan de copias de seguridad 🟢

Necesito un plan de backups para mi portátil personal. Dame una checklist con: qué carpetas copiar (Documentos, Fotos, Configuración), dónde guardarlas (disco externo, nube) y cada cuánto hacer cada copia. Guárdalo como plan-backups.md.

Variación: pídele una versión para un equipo con datos críticos de empresa o para un servidor.

Sugerencia: programa recordatorios periódicos para no olvidar las copias.

Documentación del sistema 🟢

Crea un inventario de mi equipo con: hardware (procesador, RAM, discos, gráfica), software instalado (SO, programas clave con versiones), licencias y configuraciones destacadas. Déjame espacio para anotar cambios. Guárdalo como inventario-sistema.md.

Variación: pídele que calcule el valor de reposición del hardware para el seguro o que genere una tabla comparativa con las especificaciones técnicas.

Sugerencia: mantenlo actualizado; te será muy útil para reparaciones o al vender el equipo.

Calendario de mantenimiento 🟢

Crea un plan de mantenimiento periódico para mi ordenador: limpiar archivos temporales, actualizar SO y drivers, revisar salud del disco, revisar suscripciones y limpiar el polvo interior. Indica frecuencia (semanal, mensual, trimestral). Guárdalo como mantenimiento-pc.md.

Variación: pídele que añada una columna de “última vez” para saber cuándo hiciste cada tarea.

Sugerencia: pídele recordatorios trimestrales para cada tarea de mantenimiento.

Checklist post-formateo 🟢

Acabo de formatear el ordenador e instalar el sistema desde cero. Dame una checklist ordenada de todo lo que tengo que hacer después: instalar drivers, programas esenciales, restaurar datos desde backup, configurar cuentas, personalizar el sistema. Guárdalo como post-formateo.md.

Variación: pídele versiones específicas para Windows, Linux o macOS.

Sugerencia: guarda la checklist para reutilizarla y mejorarla en futuros formateos.

Seguridad básica 🟡

`@inventario-sistema.md` A partir del inventario de mi sistema, revísame la configuración de seguridad: ¿qué servicios innecesarios podrían estar activos? ¿faltan actualizaciones críticas? ¿qué puertos o servicios suponen un riesgo? Dame una lista de acciones prioritarias.

Variación: pídele una guía de buenas prácticas de seguridad adaptada a tu perfil (uso doméstico, teletrabajo, servidor).

Sugerencia: repite esta revisión cada trimestre para mantener el sistema protegido.

/minuta 🔴

Genera un acta de reunión con estructura fija:

---
description: Genera un acta a partir de notas de reunión
---

A partir de estas notas de reunión, genera un acta con:
asistentes, orden del día, acuerdos, puntos pendientes
y fecha de próxima reunión. Usa tono formal.

$ARGUMENTS

Variación: combínalo con @ para enriquecer el acta con documentos de referencia.

Sugerencia: define una descripción clara para que OpenCode entienda bien el contexto del acta.

/liberar 🔴

Prepara la nota de release de una versión:

---
description: Prepara la nota de release de una nueva versión
---

Con los cambios de la versión `$ARGUMENTS`, genera una
nota de release con: resumen, novedades, mejoras,
correcciones y agradecimientos.

Variación: pídele que enumere los cambios respecto a la versión anterior automáticamente.

Sugerencia: combínalo con @changelog.md para generar la nota de release automáticamente.
